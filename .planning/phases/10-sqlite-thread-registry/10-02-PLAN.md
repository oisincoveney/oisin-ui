  <action>
Completely rewrite `thread-registry.ts`. Keep ALL exported types and ALL public method signatures identical. Callers must not change.

**Exported types to preserve (same shapes):**
- `ThreadStatus` — z.enum(["idle","running","error","closed"]) — DROP "unknown", no provisioning. Also update:
  - `session.ts`: change `mapAgentLifecycleToThreadStatus` fallback from `'unknown'` → `'error'`; update `toThreadSummary` return type (remove `'unknown'` from status union)
  - `packages/server/src/shared/messages.ts`: update thread status enum to remove `'unknown'`
- `ThreadLaunchConfig`, `ThreadLinks`, `ProjectRecord`, `ThreadRecord`, `ThreadRegistryState`
- `ThreadLinks` — keep `sessionKey` and `agentId` fields as `string | null | undefined` (they're still on the type for callers), but they are NOT written to DB
- `ThreadRecord.links.worktreePath` — still present on type, now sourced from `threads.worktree_path` column

**Runtime-only in-memory state (NOT in DB):**
- `agentId` — keep a `Map<string, string>` (threadKey → agentId) in memory
- `sessionKey` — keep a `Map<string, string>` (threadKey → sessionKey) in memory
- These maps are populated by `updateThread()` calls and cleared on `deleteThread()`

**Key implementation notes:**

`load()`: Run `initDb(dbPath)` if not yet initialized. Return current snapshot. No JSON file.

`getSnapshot()`: Build `ThreadRegistryState` from DB query + in-memory maps. Projects from `projects` table, threads from `threads` table joined with in-memory agentId/sessionKey maps.

`createThread(input)`:
- INSERT into threads with `status='idle'` (worktree already exists on disk at call time — thread-lifecycle.ts creates worktree first, then calls createThread)
- `links.worktreePath` is required at DB level — if `links?.worktreePath` is missing, throw an error ("worktreePath is required")
- Keep existing `createThread` signature unchanged (worktreePath passed via `links`)

`deleteThread(projectId, threadId)`: DELETE from threads WHERE project_id=? AND thread_id=?. Also clear in-memory agentId/sessionKey maps.

`updateThread(input)`: UPDATE threads SET ... WHERE project_id=? AND thread_id=?. If input.links contains agentId/sessionKey, update in-memory maps only (not DB).

`updateThreadStatus(input)`: UPDATE threads SET status=?, updated_at=?, last_status_at=? WHERE project_id=? AND thread_id=?.

`findThreadByAgentId(agentId)`: Scan in-memory agentId map, then fetch thread from DB.

`findThreadByTerminalId(terminalId)`: SELECT from threads WHERE terminal_id=?.

`setProjects(projects)`: Upsert all projects. Use INSERT OR REPLACE.

`switchThread(projectId, threadId)`: UPDATE projects SET active_thread_id=? WHERE project_id=?. Also clear active_thread_id on other projects if needed (match existing behavior — only one active thread globally).

`getActiveThread()`: SELECT projects WHERE active_thread_id IS NOT NULL, then fetch that thread.

`flush()`: No-op (SQLite writes are immediate). Keep method for interface compat.

`load()`: No-op after first call (DB already open). Keep for interface compat.

**DB row ↔ ThreadRecord mapping:**
- `launch_config` column: `JSON.stringify(launchConfig)` on write, `JSON.parse(launch_config)` on read
- `links` on ThreadRecord: `{ worktreePath: row.worktree_path, terminalId: row.terminal_id ?? null, agentId: inMemoryAgentMap.get(key) ?? null, sessionKey: inMemorySessionMap.get(key) ?? null }`

**Remove entirely:**
- All JSON file read/write logic (`readFile`, `writeFile`, `rename`)
- `THREAD_REGISTRY_FILENAME`, `THREAD_REGISTRY_VERSION` constants
- Legacy compat schema/migration code
- `LegacyCompatibilitySchema`, `LegacyPlaceholderStateSchema`
- `ThreadRegistryStateSchema` (internal, not exported)
- `"unknown"` from `ThreadStatusSchema` enum
