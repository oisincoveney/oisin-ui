The current code at ~line 275:
```ts
if (this.terminalManager && thread.links.worktreePath) {
  const ensured = await this.ensureThreadTerminalWithRetry({
    ...
    cwd: thread.links.worktreePath,
  });
```

Add a worktree path existence check BEFORE calling `ensureThreadTerminalWithRetry`:

```ts
if (this.terminalManager && thread.links.worktreePath) {
  // Validate worktree path exists on disk before attempting terminal reattach
  const { access } = await import("node:fs/promises");
  try {
    await access(thread.links.worktreePath);
  } catch {
    this.logger?.warn(
      { projectId: input.projectId, threadId: input.threadId, worktreePath: thread.links.worktreePath },
      "Worktree path missing on disk — marking thread as error"
    );
    await this.threadRegistry.updateThreadStatus({
      projectId: input.projectId,
      threadId: input.threadId,
      status: "error",
    });
    throw new Error(`Worktree path no longer exists: ${thread.links.worktreePath}`);
  }

  const ensured = await this.ensureThreadTerminalWithRetry({
    ...
  });
```

Use a static import at the top of the file instead of dynamic import if `fs/promises` isn't already imported:
```ts
import { access } from "node:fs/promises";
```

The thrown error will propagate to the session handler which already has error handling for switch failures — it will surface to the UI as a thread error state.
