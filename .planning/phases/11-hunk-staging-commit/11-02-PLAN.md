1. In diff-types.ts, add response message types to DiffSessionMessage union:
   - checkout_stage_response: { type, payload: { cwd, path, success, error, requestId } }
   - checkout_unstage_response: { type, payload: { cwd, path, success, error, requestId } }
   - checkout_commit_response: { type, payload: { cwd, success, error, requestId } }

2. In diff-store.ts, add three exported functions:

sendStageRequest(cwd: string, filePath: string): void
  - Generate requestId with randomId('stage')
  - sendWsMessage({ type: 'checkout_stage_request', cwd, path: filePath, requestId })

sendUnstageRequest(cwd: string, filePath: string): void
  - Generate requestId with randomId('unstage')
  - sendWsMessage({ type: 'checkout_unstage_request', cwd, path: filePath, requestId })

sendCommitRequest(cwd: string, message: string): void
  - Generate requestId with randomId('commit')
  - sendWsMessage({ type: 'checkout_commit_request', cwd, message, addAll: false, requestId })
  - Note: addAll: false because we only commit what's already staged

3. In handleDiffSessionMessage, add cases for checkout_commit_response:
   - On success: no action needed (diff subscription auto-refreshes)
   - On error: return error to caller (will be handled in Task 2)

Note: Stage/unstage responses don't need special handling — diff subscription refresh happens server-side.
