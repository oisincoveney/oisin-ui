  <action>
Add Push button next to Commit button in diff-panel.tsx:

1. Import ArrowUpFromLine from lucide-react (upload arrow icon)
2. Import sendPushRequest, subscribePushResponses, fetchCheckoutStatus from diff-store
3. Add state:
   - isPushing: boolean
   - pendingPushCwd: string | null
   - aheadOfOrigin: number | null (from checkout status)

4. Add useEffect to subscribe to push responses (same pattern as commit):
   - On success: toast.success("Pushed to origin")
   - On error: toast.error with message (suggest terminal for auth issues)

5. Add useEffect to fetch checkout status when cwd changes:
   - Call fetchCheckoutStatus(cwd) on mount and when cwd changes
   - Subscribe to status updates to get aheadOfOrigin

6. Add Push button in the form, after Commit button:
   ```tsx
   <Button
     type="button"
     size="sm"
     variant="outline"
     disabled={isPushing || !cwd || aheadOfOrigin === 0 || aheadOfOrigin === null}
     onClick={() => {
       if (!cwd) return
       setIsPushing(true)
       setPendingPushCwd(cwd)
       sendPushRequest(cwd)
     }}
   >
     {isPushing ? (
       <RefreshCw className="h-4 w-4 animate-spin" />
     ) : (
       <>
         <ArrowUpFromLine className="h-4 w-4 mr-1" />
         Push
         {aheadOfOrigin && aheadOfOrigin > 0 ? (
           <span className="ml-1 text-xs">↑{aheadOfOrigin}</span>
         ) : null}
       </>
     )}
   </Button>
   ```

7. Reset isPushing and pendingPushCwd on push response
8. Re-fetch checkout status after successful push to update badge

CONTEXT decisions: Button always visible, disabled when nothing to push. Badge shows ↑N count. Spinner during push. Toast for feedback.
