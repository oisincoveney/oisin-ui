<objective>
Configure the single-container Docker environment for Oisin UI.

Purpose: Provides a reliable, reproducible, isolated environment where the daemon, web UI, and terminal tools (tmux, git) can run together, with proper signal handling.
Output: A `Dockerfile`, `docker-compose.yml`, and a bootstrapper script.
</objective>

<tasks>

<task type="auto">
  <name>Task 1: Create Dockerfile</name>
  <files>Dockerfile</files>
  <action>
    Create a `Dockerfile` using `debian:bookworm` (or `node:22-bookworm`).
    Install `tini`, `tmux`, `git`, `curl`, `build-essential`. Install `bun` globally.
    Configure it to run as `root` (to simplify git worktree perms).
    Expose ports for the Daemon (e.g., 3000) and Web UI (e.g., 5173).
    Set `ENTRYPOINT ["/usr/bin/tini", "--"]` to properly reap orphaned pty processes.
  </action>
  <verify>Run `docker build -t oisin-ui-test .`.</verify>
  <done>Dockerfile builds successfully with required tools.</done>
</task>

<task type="auto">
  <name>Task 2: Setup Docker Compose</name>
  <files>docker-compose.yml</files>
  <action>
    Create `docker-compose.yml`.
    Define one service `oisin-ui` building from `.`.
    Map volumes: `/workspace` (for git repos), `/config` (for state), and `~/.ssh:/root/.ssh:ro` (for git auth).
    Map host ports to container ports (3000:3000, 5173:5173).
  </action>
  <verify>Run `docker-compose config` to validate YAML.</verify>
  <done>docker-compose.yml correctly maps volumes and ports.</done>
</task>

<task type="auto">
  <name>Task 3: Create Start Script</name>
  <files>scripts/start.sh</files>
  <action>
    Create `scripts/start.sh`.
    This script should run the daemon process and the web dev server (or static file server) concurrently (e.g., using `concurrently` or backgrounding one process and `wait`).
    Make it executable (`chmod +x scripts/start.sh`).
    Update the Dockerfile CMD to execute this script.
  </action>
  <verify>Check permissions with `ls -l scripts/start.sh`.</verify>
  <done>start.sh can launch both required processes.</done>
</task>

</tasks>

<verification>
Ensure `docker build` succeeds and `tini` is configured as the PID 1 entrypoint.
