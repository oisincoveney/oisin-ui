The problem: `handleSubscribeCheckoutDiffRequest` calls `ensureCheckoutDiffWatchTarget(cwd, compare)` which calls `computeCheckoutDiffSnapshot(cwd, compare)` which calls `getCheckoutDiff(diffCwd, ...)` which calls `requireGitRepo(cwd)` ‚Äî this throws `NotGitRepoError` when the worktree path is stale. The error propagates up and is returned as `{ error: { code, message } }` in the diff payload, surfacing "Not a git repository" in the UI.

**Fix: validate cwd before subscribing; fall back to project repoRoot if invalid.**

In `handleSubscribeCheckoutDiffRequest`, before calling `ensureCheckoutDiffWatchTarget`, check if the cwd is a valid git repo. If not, attempt to recover using the active project's repoRoot from the session's project registry.

The session already has access to a project registry (look for `this.projectRegistry`, `this.projects`, or similar ‚Äî search the session class for how projects are accessed). The fallback cwd should be the `repoRoot` of the project that owns the thread.

Concrete implementation in `handleSubscribeCheckoutDiffRequest`:

```typescript
private async handleSubscribeCheckoutDiffRequest(
  msg: SubscribeCheckoutDiffRequest
): Promise<void> {
  let cwd = expandTilde(msg.cwd)
  const compare = this.normalizeCheckoutDiffCompare(msg.compare)

  // Recover stale worktree cwd: if the requested cwd is not a git repo,
  // fall back to the active project repoRoot before subscribing.
  cwd = await this.resolveValidDiffCwd(cwd)

  this.removeCheckoutDiffSubscription(msg.subscriptionId)
  const target = await this.ensureCheckoutDiffWatchTarget(cwd, compare)
  // ... rest unchanged
}

private async resolveValidDiffCwd(requestedCwd: string): Promise<string> {
  // Fast path: cwd is valid
  try {
    await execAsync('git rev-parse --git-dir', { cwd: requestedCwd, env: READ_ONLY_GIT_ENV })
    return requestedCwd
  } catch {
    // cwd is not a git repo ‚Äî attempt recovery
  }

  // Find a project whose worktree path or repoRoot matches or contains the requested cwd,
  // and return its repoRoot as the fallback.
  // Access projects via this.projectRegistry or equivalent session field.
  // Search the session class for the field name (look for 'projects', 'projectRegistry', 'getProjects').
  const projects = this.getProjectList?.() ?? []
  for (const project of projects) {
    const repoRoot = project.repoRoot
    if (!repoRoot) continue
    try {
      await execAsync('git rev-parse --git-dir', { cwd: repoRoot, env: READ_ONLY_GIT_ENV })
      this.sessionLogger.warn(
        { requestedCwd, recoveredCwd: repoRoot },
        'Diff cwd is not a git repo; recovered to project repoRoot'
      )
      return repoRoot
    } catch {
      // this project's repoRoot is also invalid ‚Äî try next
    }
  }

  // No valid fallback found ‚Äî return original cwd and let the error surface normally
  return requestedCwd
}
```
