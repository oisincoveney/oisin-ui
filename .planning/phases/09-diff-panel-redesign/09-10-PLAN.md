In `packages/server/src/server/session.ts`, change `resolveValidDiffCwd` signature to accept optional thread identity and use it for thread-scoped fallback:

```ts
private async resolveValidDiffCwd(
  requestedCwd: string,
  hint?: { projectId?: string; threadId?: string }
): Promise<string>
```

New recovery order:
1. Fast path: `requestedCwd` is a valid git repo → return it (unchanged).
2. Thread-scoped fallback (when `hint.projectId` provided):
   - Look up the project by `hint.projectId` via `this.threadRegistry.getProject(hint.projectId)` (or equivalent list+find).
   - If the project has a valid `repoRoot` that passes `git rev-parse --git-dir`, log a warning and return it.
3. Global fallback (existing loop over all projects) — kept as final safety net.
4. Return `requestedCwd` if nothing works.

In `handleSubscribeCheckoutDiffRequest`, pass the hint:

```ts
const cwd = await this.resolveValidDiffCwd(expandTilde(msg.cwd), {
  projectId: msg.projectId,
  threadId: msg.threadId,
})
```

Do NOT change the diff target cache key `(cwd, compare)` — after recovery, each thread's cwd will resolve to its own project repoRoot, so distinct projects will naturally produce distinct keys.
