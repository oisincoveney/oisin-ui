In `packages/server/src/server/session.ts`, add a `resolveValidTerminalCwd` helper (near the existing `resolveValidDiffCwd`):

```ts
private async resolveValidTerminalCwd(
  requestedCwd: string,
  hint: { projectId: string; threadId: string }
): Promise<string> {
  // Fast path: path exists on filesystem
  if (existsSync(requestedCwd)) {
    return requestedCwd
  }

  this.sessionLogger.warn(
    { requestedCwd, ...hint },
    'Thread worktreePath does not exist; recovering terminal cwd to project repoRoot'
  )

  // Thread-scoped fallback: use owning project repoRoot
  const projects = await this.threadRegistry.listProjects()
  const project = projects.find((p) => p.projectId === hint.projectId)
  if (project?.repoRoot && existsSync(project.repoRoot)) {
    return project.repoRoot
  }

  // Last resort: return original (terminal will surface the error naturally)
  return requestedCwd
}
```

`existsSync` is already imported from `node:fs` or add the import. Use the same `READ_ONLY_GIT_ENV` / `execAsync` pattern already present if you prefer a git-repo check, but `existsSync` is sufficient and faster.
