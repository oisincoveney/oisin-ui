**Problem A — Dual TerminalView branches remount on panel toggle:**
The current JSX has two branches:
```tsx

  <ResizablePanelGroup ...>

  </ResizablePanelGroup>

```
When `diffPanelOpen` flips, React unmounts one TerminalView and mounts a different one, triggering full xterm init/dispose.

Fix: Keep a single `<TerminalView>` always mounted. Use CSS to show/hide the resizable layout wrapper around it. Restructure so TerminalView is always in the tree but the ResizablePanelGroup wraps it only when needed — achieved by rendering TerminalView outside the conditional and using absolute/relative positioning or a stable container.

Concrete approach: Render TerminalView once in a stable container. Use a wrapper div that is always present. When diff panel is open (desktop), show the ResizablePanelGroup with the diff panel alongside; when closed, show just the terminal. The key is TerminalView itself must NOT be inside the conditional branch.

One clean pattern: render TerminalView in a `<div className="absolute inset-0">` that is always mounted, and overlay/resize it using CSS when the diff panel is open. The ResizablePanel for the terminal can use `style={{ display: diffPanelOpen ? undefined : 'none' }}` on the diff panel side, but the TerminalView div must remain in the DOM.

**Problem B — Unstable callback identity causes TerminalView effect re-run:**
`handleTerminalReady` and `handleResize` are defined as plain functions inside the component body. TerminalView's `useEffect` depends on `[onTerminalReady, onDispose]` — every render creates new function references, causing the effect to re-run and xterm to reinitialize.

Fix: Wrap `handleTerminalReady` and `handleResize` in `useCallback` with stable deps. Since these functions close over refs (not state), they can use `useCallback(..., [])` (empty deps — all accessed values are refs).

```tsx
const handleTerminalReady = useCallback((term: Terminal) => {
  // existing body unchanged
}, [])

const handleResize = useCallback((cols: number, rows: number) => {
  // existing body unchanged  
}, [])
```

Both functions only access refs (`terminalRef`, `adapterRef`, `terminalIdRef`, `sendWsBinary`, etc.) and state setters — all stable. Empty dep array is correct.

Add `useCallback` to the import from 'react'.

<task type="auto">
  <name>Task 2: Decouple TerminalView init effect from callback prop identity</name>
  <files>packages/web/src/terminal/terminal-view.tsx</files>
  <action>
The `useEffect` in TerminalView has `[onTerminalReady, onDispose]` in its dependency array. Even after memoizing in App.tsx, this is fragile — any future caller that forgets `useCallback` will cause churn.

Fix: Use refs to bridge the callbacks so the effect dependency array can be `[]` (run once on mount):

```tsx
export function TerminalView({ onTerminalReady, onDispose, onResize, className = '' }: TerminalViewProps) {
  const containerRef = useRef<HTMLDivElement>(null)
  const terminalRef = useRef<Terminal | null>(null)
  const fitAddonRef = useRef<FitAddon | null>(null)
  const [terminalState, setTerminalState] = useState<Terminal | null>(null)
  
  // Bridge callbacks through refs so the init effect doesn't re-run on identity change
  const onTerminalReadyRef = useRef(onTerminalReady)
  const onDisposeRef = useRef(onDispose)
  useEffect(() => { onTerminalReadyRef.current = onTerminalReady }, [onTerminalReady])
  useEffect(() => { onDisposeRef.current = onDispose }, [onDispose])

  useTerminalResize({ terminal: terminalState, fitAddon: fitAddonRef.current, container: containerRef.current, onResize })

  useEffect(() => {
    if (!containerRef.current) return
    // ... existing terminal setup unchanged ...
    setTimeout(() => {
      try {
        fitAddon.fit()
        onTerminalReadyRef.current?.(term)  // use ref, not prop directly
      } catch { /* ignore */ }
    }, 10)

    return () => {
      onDisposeRef.current?.()  // use ref
      webglAddon?.dispose()
      term.dispose()
      terminalRef.current = null
      fitAddonRef.current = null
      setTerminalState(null)
    }
  }, [])  // empty deps — runs once on mount
  
  return <div ref={containerRef} className={`w-full h-full overflow-hidden ${className}`} style={{ padding: '8px' }} />
}
```

This is the canonical React pattern for stable effects with changing callbacks. The effect runs once; callbacks are always current via refs.
