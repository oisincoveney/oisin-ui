<objective>
Split the flat `files` array in the diff payload into `stagedFiles` and `unstagedFiles` by running `git diff --cached` (staged) and `git diff` (unstaged) separately in the daemon, then propagate the split through the shared message schema, server session, and web diff-store types.

Purpose: The UI needs separate staged/unstaged arrays to render two collapsible sections. Currently the daemon runs only `git diff HEAD` which conflates staged and unstaged changes.

Output: `checkout_diff_update` payload carries `stagedFiles: ParsedDiffFile[]` and `unstagedFiles: ParsedDiffFile[]`. The web diff-store caches both. The existing `files` field is kept for backward compat (set to `[...stagedFiles, ...unstagedFiles]`).
</objective>
