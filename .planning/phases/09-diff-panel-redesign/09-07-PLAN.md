Two targeted fixes in App.tsx:

**Fix A — Pass updatedAt to DiffPanel:**

`activeDiffEntry` already has `updatedAt: string` (set in `toCacheEntry` in diff-store.ts). App.tsx currently does NOT pass `updatedAt` to `<DiffPanel>`. DiffPanel's `formatUpdatedAt` returns "Waiting for diff snapshot" when `updatedAt` is null/undefined.

In the JSX where `<DiffPanel>` is rendered, add:
```tsx
updatedAt={activeDiffEntry?.updatedAt ?? null}
```

DiffPanel already accepts `updatedAt?: string | null` — no component changes needed.

**Fix B — Guard delayed scroll-to-bottom against disposed terminal:**

`scheduleScrollToBottom` sets a 250ms timeout that calls `terminalRef.current?.scrollToBottom()`. If the terminal is disposed during those 250ms (e.g. during remount churn), `scrollToBottom()` throws because xterm's internal buffer is gone.

The fix: pass an `onDispose` callback to TerminalView that clears the pending scroll timer when the terminal is disposed. In App.tsx:

```tsx
const handleTerminalDispose = useCallback(() => {
  clearPendingScroll()
  terminalRef.current = null
}, [])
```

Then pass it to TerminalView:
```tsx
<TerminalView
  onTerminalReady={handleTerminalReady}
  onDispose={handleTerminalDispose}
  onResize={handleResize}
  className="h-full w-full"
/>
```

Note: `handleTerminalReady` already sets `terminalRef.current = term`. The dispose handler clears it and cancels any pending scroll. This prevents the stale ref from being called after disposal.

Also: `scheduleScrollToBottom` should guard the inner call:
```tsx
const scheduleScrollToBottom = () => {
  clearPendingScroll()
  pendingScrollToBottomRef.current = setTimeout(() => {
    pendingScrollToBottomRef.current = null
    try {
      terminalRef.current?.scrollToBottom()
    } catch {
      // terminal may have been disposed between schedule and execution
    }
  }, 250)
}
```

Add `useCallback` wrapping to `handleTerminalDispose` (import already updated in plan 06).
